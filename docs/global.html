Object

AdminBroOptions

AdminBro takes a list of options of the entire framework. All off them have default values, but you can easily tailor them to your needs

Properties:
Name Type Attributes Default Description
rootPath String <optional>
 '/admin'

under which path AdminBro will be available
logoutPath String <optional>
 '/admin/logout'

url to a logout action
loginPath String <optional>
 '/admin/login'

url to a login page
databases Array.<BaseDatabase> <optional>
 []

array of all databases
resources Array.<Object> <optional>
 []

array of all database resources. Resources can be given directly or nested within an object along with its options
resources[].resource BaseResource <optional>

class, which extends BaseResource
resources[].options ResourceOptions <optional>

options for given resource
dashboard Object <optional>

your custom dashboard page
handler BaseAction.handler <optional>

action handler which will override default dashboard handler - you can perform actions on the backend there and pass results to component
component Component <optional>

Component which will be rendered on the dashboard
version Object <optional>

sets the versions visibility
admin Boolean <optional>

if set to true, shows current AdminBro version
app String <optional>

if set, shows this version in the UI
branding Object <optional>

branding settings
logo String <optional>

logo shown in AdminBro in the top left corner
companyName String <optional>

company name
theme Object <optional>

override custom css properties as colors and paddings. See This file example: colors: {primary: 'red'}
softwareBrothers Boolean <optional>

if Software Brothers logos should be shown in the sidebar footer
assets Object <optional>

assets object
styles Array.<String> <optional>

array with a paths to styles
scripts Array.<String> <optional>

array with a paths to scripts
globalsFromCDN Array.<String> <optional>
 true

indicates if globals like React, ReactDOM etc. should be taken from CDNs. If set to false, local bundle file will be used (makes sense with slower internet connection)
env Object.<String, String> <optional>

environmental variables passed to the frontend

const AdminBro = require('admin-bro')
//...
const adminBro = new AdminBro({
  rootPath: '/xyz-admin',
  logoutPath: '/xyz-admin/exit',
  loginPath: '/xyz-admin/sign-in',
  databases: [connection]
  resources: [{ resource: ArticleModel, options: {...}}]
  branding: {
    companyName: 'XYZ c.o.'
